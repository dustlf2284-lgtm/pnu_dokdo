# 범용 Information Architecture (IA) 작성 스타일 가이드

이 문서는 웹/앱 프로젝트의 코드, 프로토타입, 화면 흐름을 분석하여 도출한 IA를 Mermaid.js 마인드맵 코드와 JSON 형태로 변환할 때 AI 에이전트가 반드시 준수해야 하는 범용 작성 기준입니다. 대상 서비스의 규모나 복잡도에 상관없이 일관된 시각화 결과를 보장합니다.

## 1. 유연한 계층(Depth) 구조 원칙
앱마다 계층 구조가 다르므로, 고정된 Depth 기준을 두지 않고 논리적인 포함 관계에 따라 **N-Depth**로 무한 확장할 수 있도록 작성합니다.

* **최상위 (Root):** 프로젝트명 또는 앱 이름 (`(( ))` 괄호 사용)
* **대분류 (Platform/Role):** 사용자/관리자, Web/App 등 서비스 주체가 나뉠 경우에만 사용
* **네비게이션 분류 (Category):** GNB, LNB, 하단 탭바 등 화면을 묶는 개념적인 폴더 (기호 없이 텍스트만 작성)
* **단말 노드 (Leaf Node):** 실제 사용자가 마주하는 화면, 팝업, 모달 등 (하단 기호 규칙 적용)

## 2. Mermaid.js 시각화 식별 기호 (핵심 규칙)
단순한 텍스트 나열을 넘어, SVG 변환 시 노드의 형태만 보고도 UI 요소의 성격을 파악할 수 있도록 **반드시 Mermaid 도형 문법을 적용**합니다.

* **[화면/페이지]:** 실제 화면 전환이 일어나는 풀 페이지는 대괄호 `[ ]` 로 감쌉니다.
  * *예시:* `[메인 홈 화면]`, `[상품 상세 페이지]`
* **(모달/팝업):** 화면 이동 없이 기존 화면 위에 뜨는 중앙 팝업이나 모달은 소괄호 `( )` 로 감쌉니다.
  * *예시:* `(결제 완료 팝업)`, `(권한 요청 모달)`
* **{{바텀시트/드로어}}:** 화면 하단에서 올라오거나 측면에서 밀려 나오는 UI는 중괄호 2개 `{{ }}` 로 감쌉니다.
  * *예시:* `{{옵션 선택 바텀시트}}`, `{{햄버거 사이드 메뉴}}`
* **카테고리명 (폴더):** 화면이 아니라 하위 화면들을 묶어주는 논리적 단위는 기호 없이 작성합니다.
  * *예시:* `1. 마이페이지`, `고객센터`

## 3. 들여쓰기(Indentation) 및 문법 엄수
Mermaid 마인드맵은 들여쓰기로 부모-자식 관계를 판단합니다.
* 하위 계층으로 내려갈 때마다 **반드시 스페이스(Space) 2칸**씩 일관되게 추가합니다. (Tab 사용 절대 금지)
* JSON의 `mermaidCode` 필드 내에서는 줄바꿈을 이스케이프 문자(`\n`)로 처리해야 합니다.

## 4. 동적 라우팅 및 상태 분기 처리
* **동적 라우팅 (Dynamic Routes):** 코드 상의 `/user/:id` 또는 `[id]` 형태는 추상적인 변수명을 버리고 직관적인 화면명으로 해석하여 작성합니다. (예: `[유저 프로필 상세 화면]`)
* **상태에 따른 분기:** 로그인 여부, 권한 등에 따라 진입하는 화면이 다르거나 분기되는 경우 텍스트 내에 꺾쇠 `< >` 로 조건을 명시합니다.
  * *예시:* `<비로그인 시> [로그인 화면]`, `<로그인 시> [내 정보 화면]`

## 5. 공통 / 예외 흐름 분리
특정 메뉴 하위에 종속되지 않고 앱 전반에서 접근 가능한 화면(예: 스플래시, 전역 검색, 설정, 404 에러 화면 등)은 특정 네비게이션 하위로 억지로 묶지 않고, Root 하위에 '공통 영역' 등의 별도 카테고리를 빼서 분류합니다.
